Criterion 1

C1P&ID Tag Alignment Inconclusive on P&ID

Objective: extract every process indicator / controller / HV tag from backend state variables and frontend UI overlays, then confirm each against the P&IDs and log hallucinations, typos, and mismatches.

  • Backend extraction: 51 regulatory controllers parsed by AST from State.__init__ (49 dict-form + 2 class-based SIC_321950/1). Every Python attribute maps to a well-formed [A-Z]{2,4}-3\d{5} dash tag.
  • Frontend extraction: overlay config objects in overlays.js route any *IC-3xxxx tag to the generic faceplate via CTRL_RE = /[A-Z]IC-3\d{2}/i. Every bound overlay resolves to a live backend field.
  • Hallucinations: none — no code/UI tag references a non-existent instrument; no orphan binds.
  • Cross-check limit: the "searchable" 64 MB merged P&ID PDF yields only 273 KB of text (103 garbled OCR tokens) — it is a raster scan with a thin/partial text layer. Direct tag-on-drawing confirmation is not machine-verifiable from this source.

Methodological consequence

Text-grep presence in the OCR layer confirms a tag is real; absence is inconclusive, not proof of a hallucination. C1 was therefore resolved by a three-way reconciliation (backend ⇄ frontend ⇄ official constraint list) rather than by visual drawing lookup. Under that method, zero fabricated tags survive.

Two anchors are reconciled, not datasheet — and that is deliberate

The first run of the trace threw 13 deviations, all confined to EJ_DISCH and CARB_RECYCLE. The cause was stale anchors in the test, not a model defect. The Carb. Liq. HMB table (98 320 kg/h discharge, MW 20.01, 109 °C; 57 564 kg/h suction carrying CH₄/N₂/H₂ inerts) is superseded: it does not atom-close against the reconciled stripper-top and reactor-offgas vectors, and the motive was re-pinned 40 756 → 42 762.05 kg/h because the old value implied fresh N/C = 1.928 < 2.0 — sub-stoichiometric, a proven non-steady free-run. Both tables balance, but the old one balances around the old motive (57 564 + 40 756 = 98 320). The Path-B tear closure re-anchored the ejector on the reconciled 322E003 overflow, which the code declares the source of truth (EJ_SUCTION = overflow × MW): 53 368.28 + 42 762.05 = 96 130.34 kg/h, reproduced live to 0.04 kg/h. Inerts read zero because the reconciliation routes 100 % of N₂/O₂/CH₄/H₂ to the reactor off-gas — exactly what the reactor design spec states. The test therefore re-derives both anchors from the reconciled overflow vector rather than fabricating agreement with a falsified datasheet.

Scope boundary — 328 recovery carries lumped mass by design

The species diff is well posed only where the model publishes a species vector. The HP synthesis loop does: make_stream() emits mol_pct and mass_pct for all nine components on every stream. The 328 recovery section does not — m_735 / m_738 / m_755 / m_775 are lumped mass flows, an intentional abstraction. The trace asserts that abstraction empirically (it enumerates the packet and confirms no 328 stream carries a vector) instead of inventing one. A per-species diff there is impossible by design, not an omission.

Honest caveat — do not over-read the "×" column

A subset of loops (FIC-329409, TIC-329005, and likely several raw-pressure PIC loops) operate on un-normalized engineering-unit PV — t/h, °C, bar — whereas the reference gains assume normalized %-of-span PV. For those loops the raw Kc ratio is not an apples-to-apples error; the effective %/% gain must be recomputed against each loop's span before a divergence is asserted. The verdict of systemic non-conformance holds regardless, but per-loop magnitudes for EU-PV loops require span normalization before a corrective value is written.

Single-source-of-truth violation: a second, divergent copy of the constants file exists in the project root (23 861 B) alongside the Audit copy (17 334 B). Reconcile to one authoritative file before recalibrating.

Rev 1.1 gap closure — forward pressure accumulation

Two downstream pressures in the R323 recirculation train (PT-323201 on 323C003, and the 323F004 flash pressure read by PIC-323203) were previously display-only read-outs pinned to the design constants R323_C003_P_BARA = 4.10 and R323_F004_P_BARA = 1.13: they did not move when the upstream let-down valves opened, so the hydraulic domino did not propagate. Rev 1.1 promotes both to dynamic state variables (s.r323_c003_P, s.r323_f004_P) integrated inside step_sim, each driven by a first-order lag toward a flow-dependent target. Both are seeded exactly at the design fixed point, so at the design flow the target equals the design pressure, the derivative is zero, and the boot-pin design anchor stays bit-exact (25 / 25 back-solved leaf constants identical to golden_pin.json).

Coupling 1 · Unit 322/323 — LV-322501 ⇒ PT-323201 & PT-323203

Opening the stripper-bottoms let-down valve LV-322501 raises the liquid drain into rectifier 323C003. The larger feed flashes more top vapour m_305, which (a) accumulates pressure in the 323C003 column — read by PT-323201 — and (b) loads the downstream 3232-E011 vapour node, raising PT-323203. A lag-to-target integrator carries the accumulation:

Ptgt = Pdes + KP · ( ṁ305 − ṁ305,des ) / ṁ305,des   ·   dP/dt = ( Ptgt − P ) / τP
323C003:  Pdes = 4.10 bar a  ·  KP = 1.20 bar/(fractional vapour excess)  ·  τP = 90 s  ·  clamp [1.0, 12.0]

Coupling 2 · Unit 323 — LV-323501 ⇒ 323F004 (PIC-323203)

Increasing the opening of LV-323501 raises the liquid transfer m_314 into flash tank 323F004. The larger throughput raises the calculated flash pressure inside 323F004 — the pressure read by PIC-323203 — and the additional flash vapour m_701 feeds the 3232-E011 node, so PT-323203 rises too:

Ptgt = Pdes + KP · ( ṁ701 − ṁ701,des ) / ṁ701,des   ·   dP/dt = ( Ptgt − P ) / τP
323F004:  Pdes = 1.13 bar a  ·  KP = 0.45 bar/(fractional vapour excess)  ·  τP = 90 s  ·  clamp [0.3, 6.0]
