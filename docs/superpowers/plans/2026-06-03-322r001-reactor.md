# 322R001 HP Urea Reactor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the 322R001 HP urea reactor as a reduced calibrated split-fraction model that pins its two product streams (liquid overflow → 322E001, gas off-gas → 322E003) to the shared design HMB, couples its live overflow back into the existing HP stripper feed, registers all reactor transmitters, and exposes both products in the clickable stream inspector.

**Architecture:** New pure function `react_322r001(hpcc, co2_feed_th, hic_322605_pct)` in `backend/main.py` computes pinned product vectors `ṅᵒᵛ_i = νᵒᵛ_{i,des}·s·(φ/φ_des)` and `ṅᵒᵍ_i = νᵒᵍ_{i,des}·s` (s = CO₂ throughput ratio, φ = HV-322605 opening fraction), plus a non-injected mass-closure diagnostic `closure_resid`. Because the synthesis loop is a recycle (hpcc→reactor→stripper→hpcc) and `step_sim` already runs `stripper_322e001` **before** `hpcc_322e002`, the reactor-overflow→stripper link is a **tear stream**: the reactor's overflow is stored in `state` and the stripper consumes the previous step's value. At the design point the stored value equals the existing frozen `STRIP_FEED207_KMOLH` constant, so the stripper output stays bit-identical (regression guard). Frontend changes are data-only: the generic stream-popup and the generic `bind`-path indicator renderer already handle any new entry.

**Tech Stack:** Python 3 (stdlib only, plain-`assert` self-tests — repo has no pytest), vanilla JS frontend verified with `node --check`. Windows / PowerShell; run Python with `python`.

```python
def react_322r001(hpcc: dict, co2_feed_th: float, hic_322605_pct: float) -> dict:
    """322R001 HP urea reactor (reduced calibrated split-fraction, pinned to design HMB).

    Products are pinned to the shared design HMB and scaled by throughput + valve opening:
        overflow_i = REACT_OVERFLOW_DES[i] · s · (φ/φ_des)      (liquid -> 322E001 via HV-322605)
        offgas_i   = REACT_OFFGAS_DES[i]   · s                  (gas    -> 322E003)
    where s = co2_feed_th / (CO2_DES_KGH/1000) and φ = hic_322605_pct/100.
    Reaction extents (for diagnostics): ξ_U = REACT_XI_UREA_DES·s, ξ_B = REACT_XI_BIU_DES·s.
    closure_resid = Σ feed − (Σ overflow + Σ offgas) − ξ_U  is reported, never injected.
    """
    s   = co2_feed_th / (CO2_DES_KGH / 1000.0)          # CO₂ throughput ratio (= stripper co2_scale)
    phi = hic_322605_pct / 100.0
    phi_des = REACT_HIC605_DES_PCT / 100.0
    overflow = {k: REACT_OVERFLOW_DES.get(k, 0.0) * s * (phi / phi_des) for k in MW_COMP}
    offgas   = {k: REACT_OFFGAS_DES.get(k, 0.0) * s for k in MW_COMP}
    xi_urea  = REACT_XI_UREA_DES * s
    xi_biu   = REACT_XI_BIU_DES * s
    feed     = hpcc["feed_kmolh"]
    closure_resid = (sum(feed.values())
                     - (sum(overflow.values()) + sum(offgas.values()))
                     - xi_urea)
    return {
        "overflow_kmolh": overflow,
        "offgas_kmolh":   offgas,
        "feed_kmolh":     feed,
        "xi_urea":        xi_urea,
        "xi_biu":         xi_biu,
        "closure_resid":  closure_resid,
        "T_overflow":     REACT_OVERFLOW_T_C,
        "T_offgas":       REACT_OFFGAS_T_C,
        "P_bara":         REACT_P_BARA,
        "P_offgas":       REACT_OFFGAS_P_BARA,
        "phi":            phi,
        "phi_des":        phi_des,
        "co2_scale":      s,
    }
```
